Aprende las bases de Docker y containerización de forma sencilla

¿Qué es Docker?

Docker es una plataforma de contenedores que permite empaquetar aplicaciones y sus dependencias en contenedores ligeros y portátiles. Esto resuelve el problema de "funciona en mi máquina" al asegurar que la aplicación se ejecute de la misma manera en cualquier entorno.

Conceptos Fundamentales

🐳 Contenedores

Los contenedores son unidades ligeras y ejecutables que incluyen todo lo necesario para ejecutar una aplicación: código, runtime, herramientas del sistema, bibliotecas y configuraciones.

📋 Imágenes

Las imágenes Docker son plantillas de solo lectura que definen la estructura del contenedor. Son como las "clases" de los contenedores, mientras que los contenedores son las "instancias".

📄 Dockerfile

Un Dockerfile es un archivo de texto que contiene instrucciones para construir una imagen Docker. Define el entorno, dependencias y comandos necesarios para tu aplicación.

🏭 Docker Hub

Docker Hub es el registro público de imágenes Docker donde puedes encontrar miles de imágenes listas para usar y compartir las tuyas propias.

¿Qué son los Contenedores?

Los contenedores son una tecnología de virtualización a nivel de sistema operativo que permite empaquetar una aplicación junto con todas sus dependencias en una unidad liviana, ejecutable e independiente.

🖥️ Máquinas Virtuales vs Contenedores

Máquinas Virtuales

  • Sistema operativo completo
  • Mayor consumo de recursos
  • Tiempo de arranque lento (minutos)
  • Aislamiento por hardware virtualizado
  • Mayor tamaño (GB)
VS

Contenedores

  • Comparte el kernel del SO
  • Menor consumo de recursos
  • Tiempo de arranque rápido (segundos)
  • Aislamiento por namespace
  • Menor tamaño (MB)

Casos de Uso de Docker

💻

Desarrollo Local

Crear entornos consistentes para desarrolladores. Todos trabajan con las mismas versiones de herramientas, bases de datos y servicios.

Ejemplo: Equipo desarrollando con Node.js 18, PostgreSQL 14, Redis 7
🧪

Testing y CI/CD

Automatizar pruebas en pipelines. Cada build ejecuta en un entorno limpio y reproducible.

Ejemplo: GitHub Actions, Jenkins, GitLab CI ejecutando tests en contenedores
🌐

Microservicios

Descomponer aplicaciones monolíticas en servicios independientes, cada uno en su propio contenedor.

Ejemplo: API REST, servicio de autenticación, procesador de pagos
☁️

Cloud y DevOps

Despliegue en cloud providers como AWS, Google Cloud, Azure usando Kubernetes o servicios manejados.

Ejemplo: EKS (AWS), GKE (Google), AKS (Azure)
🔄

Escalabilidad

Escalar aplicaciones horizontalmente mediante orquestadores como Docker Swarm o Kubernetes.

Ejemplo: Auto-scaling basado en CPU/memoria, load balancing
🛠️

Herramientas y Utilidades

Ejecutar herramientas sin instalarlas localmente: bases de datos, servidores web, utilidades de desarrollo.

Ejemplo: PostgreSQL, Redis, Nginx, herramientas de monitoreo
🚀

Despliegue Portátil

La misma aplicación funciona igual en desarrollo, staging y producción sin modificaciones.

Ejemplo: "Build once, run anywhere"
🔒

Seguridad y Aislamiento

Aislamiento entre aplicaciones, control de recursos, y procesos más seguros que ejecutar directamente en el host.

Ejemplo: Limitación de CPU/memoria, redes aisladas

Comandos Básicos de Docker

docker --version

Verifica la instalación de Docker

docker run hello-world

Ejecuta el contenedor de prueba "hello-world"

docker build -t mi-app .

Construye una imagen desde un Dockerfile

docker images

Lista todas las imágenes disponibles

docker ps

Muestra contenedores en ejecución

docker stop <container_id>

Detiene un contenedor específico

Ejemplo: Dockerfile para esta aplicación Angular

Aquí tienes un ejemplo de Dockerfile optimizado para aplicaciones Angular:

# Etapa 1: Construcción
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

# Etapa 2: Servidor web
FROM nginx:alpine
COPY --from=builder /app/dist/angular-docker /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

Explicación del Dockerfile:

  • FROM node:18-alpine: Usa Node.js 18 como base para la construcción
  • WORKDIR /app: Establece el directorio de trabajo
  • COPY package*.json ./: Copia archivos de dependencias
  • RUN npm ci: Instala dependencias
  • RUN npm run build: Construye la aplicación Angular
  • FROM nginx:alpine: Segunda etapa usando Nginx para servir archivos
  • EXPOSE 80: Expone el puerto 80

Mejores Prácticas

✅ Usa imágenes base minimalistas

Utiliza imágenes Alpine Linux que son más pequeñas y seguras que las imágenes completas.

✅ Multi-stage builds

Usa construcciones de múltiples etapas para crear imágenes más pequeñas y sin dependencias de desarrollo.

✅ .dockerignore

Crea un archivo .dockerignore para excluir archivos innecesarios como node_modules y archivos de prueba.

✅ Cache de dependencias

Copia primero los archivos package.json para aprovechar el cache de Docker en las instalaciones.
